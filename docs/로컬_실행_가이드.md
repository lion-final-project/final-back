# 로컬 실행 가이드

## 인증 실패 시 401 응답 (팀 공통)

- `SecurityLocalConfig`와 `SecurityConfig` 모두 **exceptionHandling**으로 인증 실패 시 **401 Unauthorized**를 반환하도록 설정되어 있습니다.
- 인증이 필요한 API(`/api/cart`, `/api/orders`, `/api/checkout` 등)에 토큰/쿠키 없이 요청하면 **401**이 반환됩니다.
- 팀원이 동일하게 사용하려면 두 설정에서 `.exceptionHandling(ex -> ex.authenticationEntryPoint(...))` 블록을 유지하면 됩니다.

## Connection to localhost:5432 refused 오류

이 오류는 **PostgreSQL이 실행 중이지 않을 때** 발생합니다. 애플리케이션이 `localhost:5432`로 접속을 시도하지만 DB 서버가 없어 연결이 거부됩니다.

---

## 해결 방법

### 1) Docker로 PostgreSQL 실행 (권장)

1. **Docker Desktop을 먼저 실행**합니다.  
   - "Cannot connect to the Docker daemon" / "Is the docker daemon running?" 이 나오면 Docker Desktop이 꺼져 있는 상태입니다.
2. Docker가 켜졌는지 확인한 뒤, 프로젝트 루트(`final-back`)에서 다음을 실행합니다.

   ```bash
   cd final-back
   docker compose up -d db
   ```

3. DB가 올라올 때까지 **5~10초** 기다린 뒤, 스프링 부트를 다시 실행합니다.

   ```bash
   ./gradlew bootRun
   ```

- `.env`에 설정된 값(`POSTGRES_USER=myuser`, `POSTGRES_PASSWORD=mypassword`, `POSTGRES_DB=mydatabase`, `DB_PORT=5432`)이 `application-local.yml` 기본값과 맞습니다.
- Redis/MinIO도 필요하면 `docker compose up -d` 로 한 번에 띄울 수 있습니다.

### 2) 로컬에 PostgreSQL이 설치된 경우

- PostgreSQL 서비스를 켭니다.
  - macOS (Homebrew): `brew services start postgresql@16` (또는 설치한 버전)
  - Windows: 서비스 관리자에서 PostgreSQL 서비스 시작
- 포트 `5432`에서 동작하는지 확인합니다.
- 다음에 맞게 DB/유저를 만듭니다 (또는 `application-local.yml` / 환경 변수를 이미 만든 DB에 맞춥니다).
  - 데이터베이스: `mydatabase`
  - 사용자: `myuser`
  - 비밀번호: `mypassword`

PostGIS 확장이 필요하므로, Docker 이미지 `postgis/postgis:16-3.4`를 쓰는 방법을 권장합니다.

## 백엔드 정상 동작 확인 방법 (Phase5)

1. **기동**
   - `./gradlew bootRun` (또는 `-Dspring.profiles.active=local` 로 local 프로필 적용).
   - 콘솔에 `Started ... Application` 및 **결제 더미데이터** 관련 로그(user@test.com 장바구니, 배송지, 결제수단)가 출력되면 시드 정상 적용.

2. **주문서 미리보기**
   - `GET /api/checkout?cartItemIds=1,2,3&addressId=1` (인증 필요).
   - 200 + body에 address, payment, storeGroups, priceSummary 포함되면 정상.

3. **주문 생성**
   - `POST /api/orders` (인증 필요), Body 예시:
     ```json
     {
       "addressId": 1,
       "paymentMethodId": 1,
       "deliveryTimeSlot": "11:00~14:00",
       "deliveryRequest": "문 앞에 놔주세요.",
       "cartItemIds": [1, 2, 3],
       "couponId": null,
       "usePoints": 0
     }
     ```
   - 201 + orderId, orderNumber, storeOrders, payment 등 반환되면 정상. DB의 `orders`, `store_orders`, `order_products`, `payments`에 행이 추가됨.

4. **주문 상세**
   - `GET /api/orders/{orderId}` (인증 필요).
   - 200 + order, storeOrders, payment 포함되면 정상.

5. **주문 생성 시 알림**
   - 주문 생성(POST /api/orders) 성공 시 서버에서 **결제 확정 알림**이 자동 생성됨. `notifications` 테이블에 1건 추가되고, 로그에 `[알림] 결제 완료 알림을 생성합니다` 출력.

6. **장바구니·주문서 API 인증**
   - `local` 프로필에서도 `/api/cart`, `/api/cart/**`, `/api/checkout`, `/api/checkout/**`는 **인증 필요**(Bearer 또는 쿠키). 비인증 요청 시 401.

7. **CLI로 DB 반영**
   - 애플리케이션 기동 시 `CommandLineRunner`(LocalDataInitializer)가 실행되어 시드가 DB에 반영됨. 별도 CLI 스크립트 없이 `bootRun`만으로 DB에 더미 반영됨.

## 요약

| 증상 | 원인 | 조치 |
|------|------|------|
| `Connection to localhost:5432 refused` | PostgreSQL 미실행 | Docker로 `db` 서비스 기동 또는 로컬 PostgreSQL 서비스 시작 |
| `Cannot connect to the Docker daemon` | Docker 미실행 | **Docker Desktop**을 켠 뒤 다시 `docker compose up -d db` 실행 |

Docker 사용 시: **반드시 Docker Desktop 실행 후** `docker compose up -d db` 실행하면 됩니다.
