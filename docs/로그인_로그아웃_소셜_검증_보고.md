# 로그인 / 로그아웃 / 소셜 로그인 백로그 검증 보고

> Phase1~Phase5: 구현 확인, 코딩 컨벤션, 백엔드 로그, 팀 협업, roles 반영

## Phase1: 백로그 대비 구현 확인

### 로그인 (이메일/비밀번호)

| 항목 | 상태 | 비고 |
|------|------|------|
| POST /api/auth/login 컨트롤러/서비스/DTO | ✅ | AuthController.login, AuthService.login, LoginRequest, LoginResponse |
| 이메일+비밀번호 검증, 비밀번호 암호화 비교 | ✅ | PasswordEncoder.matches, UserRepository.findByEmail |
| 사용자 상태(정지/비활성) 확인 | ✅ | user.getStatus() != UserStatus.ACTIVE → USER_STATUS_FORBIDDEN |
| 권한(roles) 로딩 | ✅ | user.getUserRoles() → Role.roleName 리스트로 반환 |
| Access 30분 / Refresh 14일 발급 | ✅ | JwtProperties accessTokenValiditySeconds(1800), refreshTokenValiditySeconds(1209600) |
| Refresh Token 저장 (refresh_tokens) | ✅ | RefreshTokenRepository.save, 로그인 시 기존 삭제 후 저장 |
| 200 OK + accessToken, refreshToken, roles 응답 | ✅ | Set-Cookie + body(LoginResponse: userId, email, name, roles, accessToken, refreshToken) |
| 이메일 없음 404 / 비번 불일치 401 / 정지 403 | ✅ | EMAIL_NOT_FOUND(404), PASSWORD_MISMATCH(401), USER_STATUS_FORBIDDEN(403) |

**프론트**: AuthModal 로그인 폼 → POST /api/auth/login (credentials: include) → 성공 시 onLoginSuccess(json.data), 404/401/403 시 메시지 분기 표시.

### 로그아웃

| 항목 | 상태 | 비고 |
|------|------|------|
| POST /api/auth/logout (Bearer 인증 필요) | ✅ | SecurityConfig에서 /api/auth/logout 은 anyRequest().authenticated() 로 인증 필요 |
| refreshToken 서버에서 제거/폐기 | ✅ | AuthService.logout → refreshTokenRepository.findByToken → delete(storedToken) |
| 200 OK + "로그아웃 되었습니다." | ✅ | ResponseEntity.ok + ApiResponse.success("로그아웃 되었습니다.") + Set-Cookie(쿠키 삭제) |
| 로그아웃 후 동일 refreshToken 재발급 불가 | ✅ | DB에서 삭제하므로 refresh 시 401 |
| 토큰 없음/만료/위조 시 401 | ✅ | REFRESH_TOKEN_MISSING(400), REFRESH_TOKEN_INVALID(401). Bearer 없으면 JWT 필터 통과 후 authenticated=false → 401 |

**프론트**: App.handleLogout → POST /api/auth/logout (credentials: include, body: {}) → setIsLoggedIn(false), setUserRole('CUSTOMER').

### 소셜 로그인 (OAuth2, 카카오)

| 항목 | 상태 | 비고 |
|------|------|------|
| OAuth 토큰 검증, 사용자 정보 조회 | ✅ | Spring OAuth2 code → token → /v2/user/me |
| social_logins 동일 provider+providerUserId → 기존 유저 로그인 | ✅ | KakaoService.isKakaoUserRegistered, findOrCreateByKakaoInfo |
| 없으면 신규 → 가입 폼 후 User+SocialLogin, CUSTOMER | ✅ | 세션 pending → POST /api/auth/social-signup/complete → completeKakaoSignup |
| 신규 필수 가입: 이메일 중복, 휴대폰 인증, 약관 | ✅ | SocialSignupCompleteRequest 검증, 서비스에서 이메일/휴대폰 중복 검사 |
| 기존 소셜 회원 로그인 시 리다이렉트 + 세션 | ✅ | ?kakao=success, 프론트에서 GET /api/auth/me 로 상태 반영 |
| 신규 가입 완료 시 201 + Set-Cookie + MeResponse | ✅ | completeSocialSignup → issueTokensForUser → Set-Cookie(accessToken, refreshToken) + MeResponse(userId, email, name, roles) |
| 신규 시 리다이렉트로 회원가입 필요, JWT 미발급 | ✅ | ?kakao=signup_required |
| OAuth 실패 401 / 이메일 중복 409 | ✅ | OAUTH_VERIFICATION_FAILED(401), DUPLICATE_EMAIL(409) |
| CUSTOMER 부여, social_logins 저장 | ✅ | completeKakaoSignup 내 user_roles, social_logins 저장 |

**프론트**: 카카오 버튼 → redirect to /oauth2/authorization/kakao; 콜백 후 kakao=success → GET /api/auth/me → handleLoginSuccess(data.roles); kakao=signup_required → AuthModal social-extra → POST social-signup/complete → handleLoginSuccess.

## Phase3: 백엔드 로그에서 확인 가능 여부

다음 로그로 플로우를 백엔드에서 추적할 수 있음.

| 구간 | 로그 위치 | 메시지 예시 |
|------|-----------|-------------|
| 로그인 성공 | AuthService.login | `[로그인] 성공 email=..., userId=..., roles=[...]` |
| 로그아웃 | AuthService.logout | `[로그아웃] 토큰 무효화 userId=...` |
| 카카오 기존 회원 | OAuth2LoginSuccessHandler | `[OAuth2 카카오] 기존 회원 로그인 처리 providerUserId=...` |
| 카카오 최초 | OAuth2LoginSuccessHandler | `[OAuth2 카카오] 최초 로그인 - 회원가입 폼 필요 providerUserId=...` |
| 소셜 가입 완료 API | AuthController | `[소셜 가입 완료] API 호출 providerUserId=...` |
| 소셜 가입 처리 | KakaoService.completeKakaoSignup | `[카카오 소셜 가입] 시작/완료`, 이메일·휴대폰 중복 시 warn |
| JWT 인증 스킵 | JwtAuthenticationFilter | log.debug (레벨 DEBUG 시) |
| BusinessException | GlobalExceptionHandler | `BusinessException: AUTH-xxx - 메시지` |

## Phase4: 팀 협업 셋팅에서 동작

- **백엔드**
  - `application-local.yml` 에서 DB, Redis, JWT, 카카오 등은 **환경 변수**로 오버라이드 가능 (예: `KAKAO_CLIENT_ID`, `KAKAO_FRONTEND_SUCCESS_URL`).
  - 포트 기본 8080. 다른 포트 사용 시 프론트 `VITE_API_BASE_URL` 과 카카오 리다이렉트 URI를 해당 호스트/포트에 맞게 설정.
- **프론트**
  - `VITE_API_BASE_URL`: 백엔드 URL (기본 `http://localhost:8080`).
  - `VITE_KAKAO_OAUTH_AUTHORIZE_URL`: 카카오 로그인 시작 URL (기본 `http://localhost:8080/oauth2/authorization/kakao`).
  - 팀원별로 `.env` 또는 `.env.local` 에 위 변수 설정 시 각자 환경에서 동일하게 동작.
- **CORS**
  - `SecurityConfig`: localhost:5173, localhost:3000, 127.0.0.1:5173, 127.0.0.1:3000 허용. 다른 프론트 포트 사용 시 해당 origin 추가 필요.

## Phase5: roles 반영 여부

| 구간 | roles 반영 |
|------|------------|
| 로그인 응답 | LoginResponse.roles (List\<String\>) ← user_roles에서 로딩 |
| JWT | JwtTokenProvider.generateToken 시 claim "roles" 저장. JwtAuthenticationFilter에서 파싱 후 ROLE_* 권한으로 설정 |
| /api/auth/me | MeResponse.roles. 세션(CustomUserDetails)일 때는 getAuthorities() → ROLE_ 제거 후 반환; JWT일 때는 DB에서 user_roles 재조회 후 반환 |
| 소셜 가입 완료 | completeKakaoSignup에서 UserRole(CUSTOMER) 저장 → issueTokensForUser에서 roles 전달 → JWT·LoginResponse에 포함 |
| 프론트 | handleLoginSuccess(userData) 시 userData.roles 사용, setUserRole(roles[0])로 화면/라우팅 분기 (CUSTOMER, STORE, RIDER, ADMIN) |

역할은 DB(user_roles) → 로그인/me 응답 및 JWT claim → 프론트 userRole 상태까지 일관되게 반영됨.
