# 로그인·로그아웃 백로그 구현 검증

## Phase 1: 백로그 구현 여부 확인

**결과: 제시된 로그인·로그아웃 백로그가 백엔드에서 모두 구현되어 있습니다.**  
(단, `/api/auth/login`은 Security에서 `permitAll`에 포함되어 있어야 비인증 사용자가 로그인할 수 있으므로, 해당 설정이 반영되었는지 확인했습니다.)

---

### 로그인 (이메일/비밀번호)

| Task | 구현 여부 | 비고 |
|------|-----------|------|
| **API 구현: POST /api/auth/login** | ✅ | `AuthController.login`, `LoginRequest`, `LoginResponse` |
| 이메일+비밀번호 입력 검증 | ✅ | `LoginRequest`: @NotBlank, @Email (email), @NotBlank (password) |
| **인증 로직** | | |
| 비밀번호 검증(암호화 비교) | ✅ | `AuthService.login` → `passwordEncoder.matches(request.getPassword(), user.getPassword())` |
| 사용자 상태 확인(정지/비활성 등) | ✅ | `user.getStatus() != UserStatus.ACTIVE` → `USER_STATUS_FORBIDDEN` (403) |
| 권한(roles) 로딩 | ✅ | `user.getUserRoles()` → `Role.getRoleName()` 리스트 |
| **토큰 발급/저장** | | |
| Access Token(30분) + Refresh Token(14일) 발급 | ✅ | `JwtTokenProvider`, `JwtProperties` (access 1800s, refresh 1209600s) |
| Refresh Token 저장(예: refresh_tokens) | ✅ | `RefreshToken` 엔티티, `refresh_tokens` 테이블, `saveRefreshToken()` |
| 응답 스펙 맞추기 | ✅ | Set-Cookie(nm_accessToken, nm_refreshToken) + body(LoginResponse: userId, email, name, roles, accessToken, refreshToken) |
| **에러 코드** | | |
| 미존재 이메일 → 404 | ✅ | `ErrorCode.EMAIL_NOT_FOUND` (NOT_FOUND, AUTH-011) |
| 비밀번호 불일치 → 401 | ✅ | `ErrorCode.PASSWORD_MISMATCH` (UNAUTHORIZED, AUTH-012) |
| 정지 계정 → 403 | ✅ | `ErrorCode.USER_STATUS_FORBIDDEN` (FORBIDDEN, AUTH-013) |

**완료 조건**

1. ✅ 올바른 이메일/비밀번호 → 200 OK + accessToken, refreshToken, roles 등 응답 (Set-Cookie + body).
2. ✅ 이메일 없음 404, 비번 불일치 401, 정지 계정 403.
3. ✅ RefreshToken은 `refresh_tokens` 테이블에 저장·관리, 재발급( refresh ) 시 DB에서 조회 후 갱신.

### 로그아웃

| Task | 구현 여부 | 비고 |
|------|-----------|------|
| **API 구현: POST /api/auth/logout** | ✅ | `AuthController.logout`, Body: `RefreshTokenRequest` (refreshToken) |
| Bearer 인증 필요 | ✅ | Security: `/api/auth/logout` 미지정 → `anyRequest().authenticated()` → 인증 필요 |
| 토큰 무효화: refreshToken 서버 저장소에서 제거 | ✅ | `AuthService.logout` → `refreshTokenRepository.findByToken()` 후 `refreshTokenRepository.delete(storedToken)` |
| 응답/에러: 성공 메시지, 토큰 누락/무효 | ✅ | 200 + "로그아웃 되었습니다.", REFRESH_TOKEN_MISSING(400), REFRESH_TOKEN_INVALID(401) |

**완료 조건**

1. ✅ 로그인 상태에서 로그아웃 요청 시 200 OK + "로그아웃 되었습니다." (Set-Cookie로 액세스/리프레시 쿠키 삭제).
2. ✅ 로그아웃 후 동일 refreshToken으로 재발급 불가 (DB에서 삭제됨).
3. ✅ Bearer/쿠키 없거나 만료·위조 시 401 (logout은 인증 필요 API이므로 403/401).

## Phase 2: 구현 코드 위치 (백엔드)

| 구분 | 위치 |
|------|------|
| **로그인** | |
| 컨트롤러 | `AuthController.login` — POST `/api/auth/login`, `@Valid LoginRequest`, Set-Cookie + `ApiResponse.success(..., LoginResponse)` |
| 서비스 | `AuthService.login(LoginRequest)` — 이메일 조회, 비밀번호 검증, 상태 검사, roles 로딩, 토큰 발급, RefreshToken 저장 |
| 요청 DTO | `auth.dto.request.LoginRequest` — email(@NotBlank, @Email), password(@NotBlank) |
| 응답 DTO | `auth.dto.response.LoginResponse` — userId, email, name, roles, accessToken, refreshToken |
| **로그아웃** | |
| 컨트롤러 | `AuthController.logout` — POST `/api/auth/logout`, `@Valid RefreshTokenRequest`, Set-Cookie(쿠키 삭제), 200 + 메시지 |
| 서비스 | `AuthService.logout(String refreshToken)` — 검증 후 DB에서 RefreshToken 삭제 |
| 요청 DTO | `auth.dto.request.RefreshTokenRequest` — refreshToken(@NotBlank) |
| **공통** | |
| RefreshToken 저장/조회 | `auth.domain.RefreshToken`, `auth.repository.RefreshTokenRepository`, `refresh_tokens` 테이블 |
| 에러 코드 | `ErrorCode`: EMAIL_NOT_FOUND(404), PASSWORD_MISMATCH(401), USER_STATUS_FORBIDDEN(403), REFRESH_TOKEN_MISSING(400), REFRESH_TOKEN_INVALID(401) |
| Security | `SecurityConfig`: POST `/api/auth/login` → `permitAll()` (비인증 접근 허용), `/api/auth/logout` → 인증 필요 |

## Phase 3: 프론트 연동 시 참고

- 엔티티/필드는 **프론트 기준**으로 맞춘다.
- 로그인 응답: Set-Cookie(`nm_accessToken`, `nm_refreshToken`) + body `data`: userId, email, name, roles, (선택) accessToken, refreshToken.
- 로그아웃: 인증 필요(Bearer 또는 쿠키), Body `{ "refreshToken": "..." }` — 서버에서 해당 RefreshToken 무효화.

---

## Phase 4: 최소 수정 사항

- **SecurityConfig**: POST `/api/auth/login`을 `permitAll()`에 포함.  
  (미포함 시 비인증 사용자가 로그인 요청 시 403 Forbidden이 발생하므로, 로그인 가능하도록 수정 반영됨.)

---

## 요약

- **로그인**: POST `/api/auth/login`, 이메일·비밀번호 검증, 상태·roles 반영, Access/Refresh 토큰 발급 및 RefreshToken DB 저장, 404/401/403 에러 매핑 완료.
- **로그아웃**: POST `/api/auth/logout`, Bearer/쿠키 인증 필요, Body의 refreshToken을 DB에서 삭제하여 무효화, 200 + 메시지 및 쿠키 삭제, 토큰 누락/무효 시 400/401 처리 완료.

## Phase2: 다른 팀원 환경 (GitHub 풀 후 사용)

- **문서**: `협업_환경_설정_및_Role_반영.md`, `토큰_JWT_검증_방법.md` 참고.
- **백엔드**: `spring.profiles.active=local`, DB/Redis/JWT(application-local.yml) 동일 설정 시 동일 동작.
- **프론트**: `.env`에 `VITE_API_BASE_URL=http://localhost:8080` (또는 해당 팀원 백엔드 URL) 설정.
- **JWT**: 로그인/회원가입 → Set-Cookie 수신, 인증 필요 API는 쿠키 자동 전송(`credentials: 'include'`). Role은 응답 `data.roles` 기준.

## Phase3: 코딩 컨벤션·에러코드·API 검토 (docs 기준)

| 항목 | 기준 (docs) | 현재 구현 | 조치 |
|------|-------------|-----------|------|
| **API URL** | `인증 API .md`: `/api/auth/*` | `/api/auth/login`, `/api/auth/logout` | ✅ 일치 |
| **에러 코드** | `인증 API .md`: AUTH-011, AUTH-012, AUTH-013, COMMON-xxx | `ErrorCode`: 동일 코드 사용 | ✅ 일치 |
| **응답 포맷** | ApiResponse 래퍼, Set-Cookie | `ApiResponse.success`/fail, Set-Cookie | ✅ 일치 |
| **임포트** | 와일드카드 금지, 미사용 제거 | `GlobalExceptionHandler`에서 미사용 `FieldError` import 제거 | ✅ 수정 반영 |
| **로그아웃 할 일** | API는 POST /api/auth/**logout** | 백로그 문구 "POST /api/auth/login" → 오타 | 문서상 로그아웃 API는 `/api/auth/logout`으로 구현됨 |

- 코딩 컨벤션 v1의 ErrorCode 예시는 `NM001` 형식이나, **인증 API .md 및 기존 명세**는 `COMMON-xxx`, `AUTH-xxx`를 사용하므로 **현재 구현 유지**(문서·프론트와 일치).
